# 03 链接与关联

## 简版

笔记创建后自动解析 `[[笔记标题]]` 语法建立双向链接。Agent 可主动建议建立语义链接，类型包括：`related` `supports` `extends` `contradicts`。

**核心原则**：创建新笔记后，搜索相关笔记，语义关联度>0.6 时建议建立链接。

---

## 明细版

### 概述

链接是 Zettelkasten 的灵魂。所有笔记通过 **双向语义链接** 形成知识网络，支持最短路径搜索、反向链接查询、知识图谱生成。

### 链接类型

| 类型 | 语义 | 示例 |
|------|------|------|
| `related` | 一般关联 | "Docker" → "Kubernetes" |
| `supports` | 支持/佐证 | "性能测试数据" → "pnpm 更快" |
| `extends` | 扩展/深化 | "Docker 基础" → "Docker 网络配置" |
| `contradicts` | 矛盾/修正 | "旧结论" → "新发现的反例" |
| `refines` | 精炼/细化 | "架构概览" → "具体模块设计" |

### 自动链接解析

笔记内容中使用 `[[笔记标题]]` 语法时，系统自动创建双向链接：

```markdown
Docker 使用 [[bridge 网络模式]] 时需要注意...
```

系统会：
1. 查找标题为 "bridge 网络模式" 的笔记
2. 如果存在，创建 `related` 类型双向链接
3. 如果不存在，跳过（不报错）

### 主动关联建议

**触发时机**：创建新笔记后，或用户提到"这个和之前说的...有关"。

**流程**：
1. 用 `zk_search_notes` 搜索可能相关的已有笔记
2. 分析语义关联度（基于关键词重叠、标签匹配、内容相似度）
3. 关联度 > 0.6 时，向用户建议建立链接
4. 用户确认后，系统内部创建链接

**回复模板**：
```
🔗 检测到关联，建议建立链接：
[新笔记] --[类型]--> [已有笔记]
是否确认？
```

### 反向链接

使用 `zk_get_backlinks(note_id)` 可查询所有指向某笔记的链接。

**用途**：
- 评估笔记重要性（被引用越多越核心）
- 发现知识盲区（高引用笔记需要维护）
- 知识图谱构建（节点 = 笔记，边 = 链接）

### 注意事项

1. **不要过度链接**：一个笔记建议 2-5 个关键链接即可
2. **语义准确**：选择最贴切的链接类型，不要都用 `related`
3. **双向验证**：链接是双向的，删除一方不影响另一方
4. **避免循环**：系统不支持笔记链接到自身
